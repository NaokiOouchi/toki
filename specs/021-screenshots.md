# 021 — App Store Screenshot 撮影手順

参照: `specs/ROADMAP.md` §2 Phase 2B
依存: spec 015 (App Sandbox) + spec 016 (OAuth) 完了 → Toki が動作する状態
ステータス: **手順書、未着手**（ユーザー作業）

App Store Connect 用のスクリーンショット撮影手順 + Toki の見せ場をどう演出するか
のガイド。最大 10 枚を撮影してアップロードする。

## 1. 目的

- App Store Connect の Screenshot 要件を満たす
- Toki の魅力（円形時計 / 24h / Calendar 連携 / Liquid Glass）が伝わる構成
- 日英両言語のストアページで使う

## 2. Apple の要件

### 2.1 macOS App Store のサイズ

以下のいずれかの **同じサイズ** で 10 枚揃える：

| 解像度 | 用途 |
|---|---|
| **2880 x 1800** | **推奨**（Retina、最も鮮明）|
| 2560 x 1600 | Retina |
| 1440 x 900 | 標準 |
| 1280 x 800 | 標準 |

→ Retina Mac で撮影なら 2880 x 1800 がそのまま使える。

### 2.2 形式

- PNG または JPG
- RGB（CMYK 不可）
- 透過 / 影なし（背景は不透明）

### 2.3 枚数

- 最大 10 枚
- 言語ごとに別途アップロード可（日本語版 + 英語版を分けても、共通でも OK）
- 最低 1 枚必須

## 3. Toki の見せどころ 10 枚構成案

| # | 内容 | 演出 |
|---|---|---|
| 1 | **メイン** — 円形時計 + 当日の予定（円弧）+ 中央時刻 | 「これが Toki」最初の印象、複数イベント表示 |
| 2 | **常時前面表示** — 他アプリ（コードエディタ等）の上に Toki が浮かぶ | 用途が一目で伝わる |
| 3 | **ホバー詳細** — 円弧上の tooltip 表示 | インタラクション可視化 |
| 4 | **イベント popover** — 円弧クリックで詳細パネル | Meet / Calendar ボタン見える |
| 5 | **重なりイベント cycle** — `2/3` indicator + scroll | 機能の独自性 |
| 6 | **24h event 背景帯** — 終日予定が時計全体に色 | 「一目で今日は特別」 |
| 7 | **次の予定表示** — 「次まで 15 分」 | データ密度の魅力 |
| 8 | **設定パネル** — テーマカラー / 透過率 / Liquid Glass | カスタマイズの豊富さ |
| 9 | **Liquid Glass** （macOS 26+） — ガラス感ある背景 | OS 最新機能対応 |
| 10 | **メニューバーアイコン + メニュー** — 接続済み、設定…等 | システム統合 |

少なくとも **#1 / #2 / #4 / #8** は必須。残りは余裕あれば。

## 4. 撮影手順

### 4.1 macOS のスクリーンショット機能

1. **`Cmd + Shift + 5`** でスクリーンショットツールバー表示
2. オプション：
   - **「選択範囲を取り込む」** or **「ウインドウを取り込む」**
   - **「シャドウを表示」** ← OFF 推奨（App Store 規定で影なし）
3. 撮影 → デフォルトでデスクトップに PNG 保存
4. ファイル名は適宜変更（例：`01-main.png`, `02-floating.png`...）

### 4.2 Retina ディスプレイの確認

- Apple > このマックについて > Display で Retina 確認
- 撮影した PNG の実サイズを「情報を見る」で確認
- 2880 x 1800 ぴったりでなくても、近いサイズなら App Store Connect が自動リサイズ可

### 4.3 Toki ウィンドウのサイズ調整

`spec 016 - aspect 固定` で min 220x260 / max 420x460 で resize 可能。
推奨：**最大サイズ近く（420x460）** で撮影 → 見やすい

### 4.4 デモ用 Google アカウントの準備

実名 / 個人予定は隠したい。**専用デモアカウント** 推奨：

選択肢：
- 既存 Google アカウントに **ダミー予定を作成**（プロジェクトミーティング、デザインレビュー、ランチ等）
- 新規 Gmail アカウント作成 → 専用デモ用

ダミー予定例（写真映え重視）：
```
09:00-10:00 朝会 (3 人)
10:30-11:30 デザインレビュー (5 人) ← Meet あり
12:00-13:00 ランチ
14:00-15:30 プロジェクトワーク
16:00-17:00 1on1
17:30-18:00 Daily wrap-up
終日: 海外出張中（背景帯になる例）
```

これだけ入れると 24h の半分以上に円弧が並んで見栄え良い。

## 5. 撮影前のチェックリスト

- [ ] Toki が最新版で起動している（メニューバーアイコン見える）
- [ ] OAuth サインイン完了（接続済み）
- [ ] デモ用 Google アカウントの予定が見える
- [ ] 設定 (テーマカラー / 透過率) はデフォルト or 見栄え良い設定に
- [ ] Display を Retina 確認
- [ ] スクリーンショットの影設定を OFF
- [ ] 他のアプリ通知をオフ（撮影中に邪魔されない）

## 6. 撮影後

### 6.1 サイズ確認

```bash
sips -g pixelHeight -g pixelWidth ~/Desktop/01-main.png
```

2880 x 1800 等の規定サイズと一致するか確認。

### 6.2 リサイズ（必要なら）

```bash
sips -z 1800 2880 ~/Desktop/01-main.png --out ~/Desktop/01-main-resized.png
```

### 6.3 軽量化（オプション）

PNG 圧縮ツール：
- ImageOptim（Mac app、無料、ロスレス）
- TinyPNG（Web、ロスあり、より小さい）

ファイルサイズ大きすぎる場合のみ実施。

## 7. アップロード（App Store Connect）

1. https://appstoreconnect.apple.com → マイ App > Toki
2. **「macOS App」→ 「1.0 配信準備中」**
3. **「macOS App スクリーンショット」** セクション
4. ドラッグ＆ドロップ or **「+」** で複数枚アップロード
5. 順序入れ替え（重要度の高い順、#1 がメイン表示）

### 7.1 言語別

- 「日本語」「英語」両方で同じスクリーンショットを使う or 別に
- 推奨：**共通**（UI 言語は別だが Toki の見せ場は同じ）
- 言語ごとに UI が違うショットを撮るなら別途

## 8. 完了条件

- [ ] 最低 4 枚（メイン / 常時前面 / popover / 設定）
- [ ] 全枚同サイズ（推奨 2880 x 1800）
- [ ] PNG 形式
- [ ] App Store Connect にアップロード完了
- [ ] 日本語版 + 英語版で表示確認

## 9. リスク・注意事項

- **個人情報写り込み**：メールアドレス / 実名 / 会社名 / 取引先名 → デモアカウントで対策
- **著作権侵害**：他社アプリのロゴ等が背景に映らないように
- **「Beta」「Test」表記**：審査落ち要因、Toki UI に含めない
- **誤訳 / typo**：英語版スクリーンショット内のテキストを念のため確認
